# PANDA KEY — Ver.2.0.0

**매크로 키 입력 프로그램**

**Tauri v2 + Next.js 15 + React 19 + TypeScript + Tailwind CSS + shadcn/ui**

키보드·마우스 키 입력(누르기/때기) 매크로. 다중 세트 + 트리거 기반 동시 혼합 실행.
추후 `통합_Ver.5.5.4` 프로젝트에 병합 예정으로 동일한 구성·스타일·형식으로 제작.

## 기능

- 키보드 및 마우스 키 입력(누르기/때기) 매크로 — 스텝 동작 4종(카드)
  - **탭(tap)** = 누르고 떼기 (누름시간 ms), **누르기(press)** = 유지, **때기(release)** = 뗌,
    **지연(delay)** = 카드 사이 대기 (ms)
  - 분리하면 키를 누른 채 다른 키 입력 가능 — 예: `⬇Ctrl  350ms  ⬇Tab  ⬆Tab  ⬆Ctrl`
  - 시간 단위 ms, 최소 0
  - **지속 누름**: 때기 없이 끝나는(눌린 채 남는) 키는 다음 반복 사이클까지 계속 눌린 상태로 지속,
    매크로 정지 시 일괄 떼짐
- **세트 편집은 전용 페이지**(`/macro/edit?id=…`) — 팝업이 아닌 페이지 전환, 변경 자동 저장
- **타임라인 스텝 편집** (게이밍 주변기기 매크로 편집기 스타일)
  - 가로 카드 배열 — 키 카드(누르기 ⬇ / 때기 ⬆ / 탭 ⬍)와 지연 카드(ms)를 사이사이 배치
  - **좌클릭 선택 · 드래그 순서변경(삽입 지점 강조 표시) · 우클릭 편집** · 빈 공간 우클릭 = 추가 메뉴
  - 지연/누름시간은 카드에서 바로 클릭 편집
- **지연시간 옵션**
  - **개별 지연**(기본) 또는 **표준 지연 사용**(모든 지연을 한 값으로 일괄 적용) 선택
  - **개별 지연 일괄수정** — 모든 지연 카드를 한 번에 지정 값으로 변경
- **입력 녹화** — 실제 키/마우스 입력을 타이밍과 함께 자동으로 스텝화
  (keydown→누르기, keyup→때기, 간격→지연 카드). **지연 고정** 옵션 시 간격을 측정 대신 지정 값으로 기록.
  ESC/정지로 종료
- 실행 모드 4종
  - **1회** — 트리거를 누르면 스텝 시퀀스를 한 번 실행
  - **누르는 동안 반복** — 트리거를 누르고 있는 동안 반복
  - **토글 (같은 키로 멈춤)** — 한 번 누르면 무한 반복, 같은 트리거 키를 다시 누르면 멈춤
  - **토글 (아무 키로 멈춤)** — 한 번 누르면 무한 반복, 아무 키나 누르면 멈춤
- 여러 개의 매크로 세트 설정
- **그룹** — 그룹이 포함할 매크로를 **선택**하는 방식(다대다: 한 매크로가 여러 그룹에 포함 가능).
  헤더 `그룹`에서 그룹 생성·이름변경·멤버 선택, 그룹 스위치로 멤버 일괄 ON/OFF
- **동일 트리거 중복 방지** — 한 트리거당 활성 세트는 하나(중복 설정 시 기존 자동 비활성화)
- **다중 실행 모드** (설정에서 선택)
  - **동시 실행**(기본): 여러 세트가 함께 실행
  - **배타 실행**: 새 매크로 시작 시 기존은 일시정지(스택), 새 매크로 종료 시
    (트리거가 눌린 상태면) 직전 매크로 재개
- 동시 혼합 실행 — A키로 1번 매크로 실행 중 B키로 2번 매크로를 함께 실행,
  B를 떼면 2번만 종료되고 A가 눌린 동안 1번은 계속 (자동 복귀)
- 트리거로 **키보드 + 마우스 버튼**(좌/우/휠/사이드 X1·X2) 모두 지정 가능
- 세트별 **트리거 키 통과/소비** 선택 (기본: 소비)
- **활성 창 제한** — 지정한 프로그램(예: game.exe)이 포그라운드일 때만 매크로 동작
  (헤더 `설정` → 활성 창 제한, `현재 창` 버튼으로 현재 프로세스명 자동 입력)
- **전체 ON/OFF 단축키** — 엔진을 켜고 끄는 전역 단축키(기본 ON=Home, OFF=End, 변경 가능)
- **반복 최소 주기(ms) 설정** — 반복 모드 사이클 하한(입력 폭주/렉 방지, 기본 16)
- **저장 / 불러오기** — 매크로 세트 + 설정을 JSON 파일로 `내보내기`/`불러오기`
  (불러오기 시 기존 교체 / 추가(병합) 선택). 다른 PC로 옮기거나 백업에 사용.
- 긴급 정지 (모든 활성 매크로 즉시 정지) — 화면 버튼 + 트레이 메뉴
- 트레이 상주 — X 클릭 시 트레이로 숨김, 백그라운드 매크로 유지

## 시작하기 (Windows)

```
실행.bat   더블클릭 → npm install + Tauri dev 실행 (네이티브 창)
```

프로덕션 빌드 (.exe 설치 파일):

```
빌드.bat   더블클릭 → src-tauri\target\release\bundle\nsis\*.exe 생성
```

브라우저 단독 실행 (Tauri 없이 UI 확인):

```bash
npm run dev   # http://localhost:3000 — UI 동작, 단 실제 입력 주입은 비활성
```

타입 체크:

```bash
npm run typecheck
```

## 동작 원리 (Rust 엔진)

- **저수준 후킹**: `WH_KEYBOARD_LL` / `WH_MOUSE_LL` 로 트리거 키의 누르기·때기를 전역 감지.
  앱 시작 시 후킹 스레드 1회 설치, `엔진 ON` 시에만 트리거를 처리.
- **입력 주입**: `SendInput`(스캔코드 우선)으로 키/마우스 주입. 게임 호환성 향상.
  - 주입 입력은 `dwExtraInfo` 서명으로 태깅 → 후킹 콜백이 자기 입력을 무시(재귀 차단).
- **모드별 실행**: 트리거 감지 시 세트별 워커 스레드가 스텝 시퀀스를 타이밍대로 실행.
  여러 세트가 독립 스레드로 동작 → 동시 혼합 실행이 자연스럽게 지원됨.
- **즉시 중지**: 워커 대기를 `Condvar` 로 구현 — 누르는 동안 반복/토글에서 트리거 해제(또는
  토글 OFF) 시 진행 중이던 대기를 즉시 깨워 **지연 없이 중지**(남은 눌림 키도 즉시 해제).
- **자동반복 무시**: 트리거 키를 누르고 있을 때 OS 가 보내는 키 자동반복(auto-repeat)을 식별해
  무시 → "반복 없음"이 1회만, 토글이 깜빡이지 않도록(첫 누름만 처리, 뗄 때 해제).
- **입력 폭주 방지**: 반복 모드 최소 사이클 주기 16ms(약 60Hz) 보장 — 지연 미설정 매크로가
  초당 수천 회 주입해 시스템이 렉 걸리는 것을 차단.
- **트리거 소비**: 세트의 `트리거 키 통과`가 꺼져 있으면 후킹에서 해당 입력을 소비(차단).

### 물리 입력 모드 (게임이 실제 키보드로 인식)

`SendInput` 으로 주입한 입력은 저수준 후킹에서 **"주입됨(injected)" 플래그**가 보입니다.
일부 게임/안티치트는 이 플래그나 raw input 으로 매크로를 구분·차단합니다.

→ **물리 입력 모드**(헤더의 `🖥 물리 입력` 토글)는 **Interception 드라이버**를 통해
**드라이버 레벨에서 키보드 입력을 주입**하므로, 게임/OS 가 **실제 물리 키보드 입력으로 인식**합니다.

설치 (1회) — **`드라이버 설치.bat` 더블클릭**:

1. `드라이버 설치.bat` 더블클릭 → UAC 동의(관리자 권한)
2. Interception 커널 드라이버 설치 + `interception.dll` 을 앱 옆에 자동 복사
3. 안내에 따라 **재부팅**
4. 앱에서 `🖥 물리 입력` 토글 ON → 상단에 “드라이버 활성” 표시되면 적용됨

- 제거: `드라이버 제거.bat` (관리자, 재부팅)
- 드라이버 파일은 `driver\` 폴더에 동봉(install-interception.exe + interception.dll).
- 드라이버가 없으면 자동으로 SendInput 으로 폴백합니다(토글은 켜지되 “미설치” 표시).
- 물리 입력은 **키보드**에 적용됩니다(마우스는 SendInput).
- 우리가 드라이버로 주입한 키는 후킹에서 최근-주입 목록으로 식별해 트리거 재발동을 막습니다.
- DLL 은 실행 파일 디렉터리/resources 도 함께 탐색하며, 프로덕션 빌드(NSIS)에 동봉됩니다.

> Interception 출처: https://github.com/oblitum/Interception

> ⚠️ 드라이버/후킹/주입은 안티치트가 탐지·차단하거나 제재 사유가 될 수 있습니다.
> 반드시 사용 환경의 약관을 확인하세요.

### 관리자 권한 (인게임 동작 필수)

대부분의 게임은 **관리자 권한**으로 실행됩니다. 이 경우 일반 권한 프로그램의 입력 주입·후킹이
Windows UIPI(권한 격리)에 의해 **무시**되어 인게임에서 매크로가 동작하지 않습니다.

→ PANDA KEY 는 `requireAdministrator` 매니페스트로 **항상 관리자 권한으로 실행**됩니다
(`src-tauri/build.rs`). 실행 시 UAC 동의 창이 뜨는 것은 정상입니다.

- 개발 실행(`실행.bat` / `tauri:dev`)도 UAC 창이 뜹니다.
- 그래도 인게임에서 안 되면: ① 게임을 관리자 권한으로 실행했는지, ② 풀스크린(전체화면) 대신
  **테두리 없는 창/창모드** 인지, ③ 안티치트가 입력을 차단하는지 확인하세요.

> ⚠️ 일부 안티치트가 적용된 게임에서는 저수준 후킹/입력 주입이 차단되거나
> 계정 제재 사유가 될 수 있습니다. 사용 환경의 약관을 확인하세요.

## 통합 병합 가이드

`통합_Ver.5.5.4` 의 도메인 이식 패턴(`lib/{domain}` + `app/{route}` + `components/{domain}`)과
동일하게 구성됨. 병합 시:

1. `lib/macro/*`, `components/macro/*`, `app/macro/*` 복사
2. `src-tauri/src/macro_engine.rs`, `keycodes.rs` 추가 + `lib.rs` 의 매크로 커맨드/`init` 호출 병합
3. `Cargo.toml` 에 `windows` 의존성, `package.json` 변경 없음(기존 의존성 호환)
4. `site-header.tsx` 의 카테고리에 `⌨ 매크로` 추가, `--cat-macro` 토큰 병합
